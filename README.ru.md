# kb-for-agents

[English version](README.md)

Плагин для Claude Code, оформляющий паттерн трёхслойной базы знаний в виде устанавливаемых навыков и слэш-команд. Направьте его на кодовую базу или архив заметок — и получите рабочий процесс с прогрессивным раскрытием информации, описанный в статье [Knowledge Base for Agents](https://www.linkedin.com/posts/ahaidukof_karpathy-%D1%84%D0%BE%D1%80%D0%BC%D0%B0%D0%BB%D0%B8%D0%B7%D0%BE%D0%B2%D0%B0%D0%BB-%D1%82%D0%BE-%D1%87%D1%82%D0%BE-%D0%BC%D1%8B-%D0%BD%D0%B0%D1%89%D1%83%D0%BF%D0%B0%D0%BB%D0%B8-activity-7446836637143130112-o920/).

## Что внутри

- **Навыки (skills)** — `kb-ingest`, `kb-query`, `kb-lint`, `kb-check-drift`, `kb-distill`, `kb-demo`, `kb-bench`. Паттерн, закреплённый в коде.
- **Команды** — `/kb-ingest`, `/kb-query`, `/kb-lint`, `/kb-check-drift`, `/kb-distill`, `/kb-demo`, `/kb-bench`. Тонкие обёртки над навыками.
- **Шаблоны** — стартовые заготовки для индексов, саммари, решений, `_root.md`, состояния. Шаги для ручной сборки см. в [templates/README.md](templates/README.md).
- **Документация** — обоснование паттерна: [концепция](docs/concept.md), [правила навигации](docs/navigation-rules.md), [стратегия линтинга](docs/lint-strategy.md).

## Установка

### Claude Code (плагин целиком)

```
/plugin marketplace add olegvg/kb-for-agents
/plugin install kb-for-agents@kb-for-agents
```

Ставит все навыки и слэш-команды, включая демо и бенчмарк на встроенном notes-corpus.

### Claude Desktop (отдельные навыки)

Claude Desktop не загружает плагины Claude Code, поэтому runtime-навыки выкладываются отдельными zip-архивами на странице [Releases](https://github.com/olegvg/kb-for-agents/releases). Скачай нужные и загрузи через UI навыков Claude Desktop:

- `kb-ingest-<version>.zip`
- `kb-query-<version>.zip`
- `kb-lint-<version>.zip`
- `kb-check-drift-<version>.zip`

К каждому релизу приложен `SHA256SUMS` — проверка через `sha256sum -c SHA256SUMS`.

`kb-distill`, `kb-demo` и `kb-bench` — только для Claude Code: distill опирается на формат plan-документов, demo/bench зависят от встроенного notes-corpus.

## Быстрый старт

### Попробовать без настройки

После установки плагина выполни:

`/kb-demo`

Откроется меню из 5 опций для встроенного синтетического корпуса
из 150 заметок (`examples/notes-corpus/`). Можно посмотреть структуру,
задать вопрос через KB, сравнить с grep-baseline, перестроить KB с нуля
или запустить полный набор бенчмарков.

Для честных цифр сравнения на фиксированных задачах:

`/kb-bench`

Прогоняет 10 бенчмарк-вопросов в двух режимах (с KB / только grep) и
печатает side-by-side таблицу. Эталонные трейсы лежат в
`benchmarks/notes-corpus/traces/`.

### Применить к своему корпусу

В любой сессии Claude Code, открытой в корне вашего корпуса:

```
/kb-ingest .
/kb-query where are fees charged?
```

После того как вы внесли изменения в корпус:

```
/kb-check-drift
/kb-lint src/billing/
```

Когда запланированная работа завершена и нужно сохранить её ключевые решения:

```
/kb-distill docs/plans/2026-03-19-1400/plan-billing-rewrite.md
```

## Как это работает

Плагин создаёт директорию `.kb/` **внутри вашего корпуса** (не внутри плагина). У неё три слоя плюс хранилище решений:

1. **Источники (sources)** — сам корпус. Не дублируется.
2. **Саммари (summaries)** (`.kb/summaries/<area>/*.md`) — проза по темам, по одному файлу на тему.
3. **Индексы (indexes)** (`.kb/indexes/*.md`) — навигация, ≤300 токенов на файл. Точка входа — `.kb/indexes/_root.md`.

Рядом: **Решения (decisions)** (`.kb/decisions/*.md`) — дистиллированные записи, извлечённые из завершённых планов. На них ссылаются саммари, которых они касаются, через секции `## Key decisions`; отдельной верхнеуровневой областью они не выносятся.

На любой запрос агент сначала читает индексы, затем спускается глубже. Если вопрос звучит как «как работает X и почему это было сделано именно так?», навигация приводит к саммари, а оттуда — по ссылкам на связанные решения. Контекст остаётся ограниченным, навигация прослеживается в трассах.

Полное обоснование — в [docs/concept.md](docs/concept.md).

## Что это

Стартовый набор: навыки, команды, шаблоны, концептуальная документация. Паттерн, оформленный так, чтобы применять его без повторного чтения статьи.

## Что это не

Это не запускаемый бенчмарк, не готовый пример корпуса, не RAG-библиотека и не универсальный фреймворк для агентов. Подтверждение эффективности — в сопутствующей статье и постах, а не в этом репозитории.

## Требования

- Claude Code.
- Корпус, который вы хотите задокументировать. Корпус с git-историей настоятельно рекомендуется — `/kb-lint` и `/kb-check-drift` оба зависят от git-истории.
