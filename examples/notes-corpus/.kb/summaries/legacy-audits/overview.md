# Legacy Audits — Overview

This domain covers AI-assisted auditing of legacy codebases: how to systematically surface undocumented behaviors, dead code, and architectural drift using LLMs. It also includes test-driven development with AI pair programming, which is often the follow-on to an audit.

The corpus includes direct tool notes (`tool-009`, `tool-020`), academic paper readings (`paper-018`, `paper-019`), and open-ended idea notes about incremental audit approaches (`idea-002`, `idea-016`). Project logs for `kb-for-agents` frequently mention "classic audit-of-legacy-codebases territory" as a repeated pattern encountered.
