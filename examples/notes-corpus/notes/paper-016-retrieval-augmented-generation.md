Just finished reading "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks" (https://arxiv.org/abs/2005.11401).

This reframed how I think about the knowledge freshness problem in retrieval systems. The ablation tables are where the real insight lives — the abstract undersells the nuance. The approach suggests a way to audit legacy code systematically rather than heuristically. I can see a direct application to how I structure context for long agent sessions. Applying this to context-window management could cut token usage by a meaningful fraction.
