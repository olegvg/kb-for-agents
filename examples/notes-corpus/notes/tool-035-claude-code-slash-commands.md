I kept putting off a proper look at Claude Code slash commands, but a failing deadline pushed me into it.

What worked: Incremental commits kept the feedback loop tight, which prevented runaway context growth. The iteration speed was noticeably faster once I stopped second-guessing the output. Keeping CLAUDE.md up to date with project-specific decisions saved me repeated explanatory preambles.

What didn't: Asking for too much in a single prompt consistently led to shallow, generic output. Asking for explanations and code in the same response reliably produced worse code.

Would recommend to anyone already comfortable with the basics — not a first step.
