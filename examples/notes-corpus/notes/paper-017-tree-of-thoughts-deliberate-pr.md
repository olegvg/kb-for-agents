Just finished reading "Tree of Thoughts: Deliberate Problem Solving with Large Language Models" (https://arxiv.org/abs/2305.10601).

The failure cases section was more useful to me than the success cases. This reframed how I think about the knowledge freshness problem in retrieval systems. This would pair well with a compiled-knowledge layer to reduce retrieval latency. I can see a direct application to how I structure context for long agent sessions. The pattern shows up in how RAG-vs-compiled-knowledge tradeoffs should actually be evaluated.
